Required in AWS Secrets Manager (backend)
Date: 2026-03-11

Global/shared
- JWT_SECRET
- DB_PASSWORD

backend-api
- TENANT_PROVISIONING_PASSWORD (if used)
- AI_GATEWAY_TOKEN
- BILLING_SERVICE_TOKEN
- SHARED_INTERNAL_SECRET
- AUDIT_SALT (if used)
- DUMMY_PASSWORD_HASH (if used)
- MEDIDATABOX_TOKEN (if Medidata integration active)

billing-api
- BILLING_SERVICE_TOKEN
- MEDIDATABOX_TOKEN (if Medidata integration active)

ai-service
- OPENAI_API_KEY
- AI_GATEWAY_TOKEN
- S3_SECRET_ACCESS_KEY (only if not using IAM Role)

login-gateway
- DUMMY_PASSWORD_HASH (if used)

Notes
- Keep all secret values out of git, local compose for production, and image build args.
- Prefer IAM Role for AWS service access to avoid static cloud credentials.
